## 🔧 التعديلات المنفذة

### 1. قاعدة البيانات (Supabase)

#### ملف الهجرة الجديد
**المسار:** `supabase/migrations/20260701000000_comprehensive_business_logic_overhaul.sql`

**الحجم:** ~800 سطر من كود SQL

**المحتويات:**

**أ) تحديث جداول الوثائق والتحصيلات**

تم إضافة الأعمدة التالية إلى جدول `policies`:
- `branch_id` - معرف الفرع (مفتاح أجنبي)
- `team_leader_id` - معرف رئيس المجموعة (مفتاح أجنبي)
- `first_year_start` - تاريخ بداية السنة الأولى (تاريخ)
- `first_year_end` - تاريخ نهاية السنة الأولى (تاريخ)
- `has_new_business_counted` - علم لتتبع احتساب الإنتاج الجديد (منطقي)

تم حذف الأعمدة التالية من جدول `policies`:
- `insurance_company` - لا حاجة لها (الشركة واحدة)
- `start_date` - تاريخ الإصدار هو تاريخ السريان
- `policy_duration` - النظام يعتمد على السنة الأولى فقط

تم إضافة الأعمدة التالية إلى جدول `collections`:
- `is_new_business` - تحديد ما إذا كانت تحصيلة إنتاج جديد (منطقي)
- `branch_id` - معرف الفرع (مفتاح أجنبي)

**ب) دوال حساب السنة الأولى**

تم إنشاء دالة `calculate_policy_first_year()` التي:
- تحسب بداية السنة الأولى من بداية شهر الإصدار
- تحسب نهاية السنة الأولى من نهاية الشهر الثاني عشر
- يتم استدعاؤها تلقائياً عند إدراج أو تحديث الوثيقة

مثال:
```
وثيقة صادرة في 15 يونيو 2026
first_year_start = 2026-06-01
first_year_end = 2027-05-31
```

**ج) دالة تحديد الإنتاج الجديد**

تم إنشاء دالة `mark_collection_as_new_business()` التي:
- تحدد ما إذا كانت التحصيلة من أول قسط (إنتاج جديد)
- تتأكد من عدم احتساب أكثر من أول قسط كإنتاج جديد لنفس الوثيقة
- يتم استدعاؤها تلقائياً عند إدراج تحصيلة جديدة

**د) View موحد للمقاييس**

تم إنشاء View باسم `unified_performance_metrics` الذي يجمع:
- معلومات التحصيلة (المبلغ، التاريخ، حالة الإنتاج الجديد)
- معلومات الوثيقة (معرف الوكيل، الفرع، رئيس المجموعة)
- معلومات السنة الأولى (تاريخ البداية والنهاية)
- حالة التحصيل ضمن السنة الأولى

هذا الـ View يوفر مصدر بيانات واحد لجميع الحسابات والتقارير.

**هـ) دوال حساب الأداء**

تم إنشاء دالتين:

1. `calculate_agent_performance(p_agent_id, p_month, p_year, p_branch_id)`
   - تحسب أداء الوكيل للشهر المحدد
   - تحسب الإنتاج الجديد والتحصيلات والإجمالي
   - تحسب معدل التحصيل (المحصل ÷ المستحق)

2. `calculate_branch_performance(p_branch_id, p_month, p_year)`
   - تحسب أداء الفرع للشهر المحدد
   - نفس الحسابات على مستوى الفرع

**و) تصحيح صلاحيات Super Admin**

تم إنشاء دالتين:

1. `is_user_super_admin(p_user_id)`
   - تتحقق من أن المستخدم هو super admin
   - تتحقق من الحالة النشطة والصلاحيات

2. `check_user_branch_access(p_user_id, p_branch_id)`
   - تتحقق من وصول المستخدم للفرع
   - تستثني super admin من القيود

**ز) تحديث RLS Policies**

تم تحديث جميع الـ Policies لـ:
- `clients` - عزل حسب الفرع مع استثناء super admin
- `policies` - عزل حسب الفرع مع استثناء super admin
- `collections` - عزل حسب الفرع مع استثناء super admin
- `targets` - عزل حسب الفرع مع استثناء super admin
- `tasks` - عزل حسب الفرع مع استثناء super admin
- `installments` - عزل حسب الفرع مع استثناء super admin
- `branches` - Super admin يرى الكل
- `audit_logs` - عزل حسب المستخدم مع استثناء super admin

**ح) تحديث البيانات الموجودة**

تم تنفيذ استعلامات UPDATE لـ:
- تحديث `first_year_start` و `first_year_end` للوثائق الموجودة
- تحديث `is_new_business` للتحصيلات الموجودة
- تحديث `has_new_business_counted` للوثائق ذات الإنتاج الجديد
- تعيين `branch_id` للوثائق والتحصيلات الموجودة

### 2. الواجهة الأمامية

#### تحديث Dashboard (`src/components/dashboard/Dashboard.tsx`)

**التغييرات الرئيسية:**

1. **إعادة هيكلة الحسابات**
   - تم استبدال جميع الحسابات لاستخدام `unified_performance_metrics`
   - تم تصحيح معدل التحصيل (المحصل ÷ المستحق × 100)
   - تم تصحيح حساب الإنتاج الجديد والتحصيلات

2. **الكروت الرئيسية (KPI Cards)**
   - إجمالي الإنتاج = الجديد + التحصيل
   - إجمالي الجديد = أول قسط فقط
   - إجمالي التحصيل = الأقساط التالية (السنة الأولى)
   - نسبة تحقيق الهدف = الإجمالي الشهري ÷ الهدف الشهري

3. **الكروت الإضافية**
   - المستهدف الشهري
   - المحقق الشهري
   - عدد الوثائق الجديدة
   - عدد العملاء الجدد
   - عدد التحصيلات
   - معدل التحصيل

4. **مؤشرات الأداء والترتيب**
   - أعلى 5 وكلاء (وكلاء فقط، بدون مديرين)
   - أقل 5 وكلاء (وكلاء فقط، بدون مديرين)
   - أعلى رؤساء مجموعات
   - أفضل مجموعات إنتاجية

5. **عزل البيانات حسب الفرع**
   - جميع المؤشرات تعتمد على الفرع النشط فقط
   - عند تغيير الفرع: تتغير جميع المؤشرات
   - منع خلط بيانات الفروع
